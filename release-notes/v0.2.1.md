# Release Notes - v0.2.1

## Features

### ANSI-Aware Table Rendering
- **Shared Table Renderer**: New `ui.Table` component with ANSI color support
- Proper column width calculation with colored text
- Consistent table formatting across all commands

### Positional Arguments
- **Simplified Syntax**: Use positional arguments instead of flags
  - `refresh nodegroup list develop` instead of `--cluster develop`
  - `refresh describe cluster develop` instead of `--cluster develop`
- Lists available clusters when argument is omitted

### Service Factory Pattern
- **Centralized Initialization**: New `initServices()` helpers reduce boilerplate
- Consistent service creation across commands

## Bug Fixes

- Fixed nil-safe logger in cluster service to prevent panics
- Fixed errcheck/staticcheck/unused linter warnings
- Improved pagination token handling in tests

## Technical Details

### New Components
- `ui.Table`: ANSI-aware table renderer with automatic column sizing
- `Outf`/`Outln` helpers for consistent stdout messages
- Service initialization factory functions

### Performance
- Optimized `truncateANSI` capacity estimation
- Batch table rendering for better output
- Faster cache key building in cluster service

## Usage Examples

```bash
# New positional syntax
refresh nodegroup list develop
refresh describe cluster production

# Old flag syntax still works
refresh nodegroup list --cluster develop
```
